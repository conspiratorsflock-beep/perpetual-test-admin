# Incident Response Runbook

## Severity Levels

### SEV1 - Critical
- Complete service outage
- Data loss or corruption
- Security breach

**Response:**
1. Page on-call engineer immediately
2. Create incident channel
3. Notify stakeholders
4. Begin rollback if applicable

### SEV2 - High
- Major feature degradation
- Significant performance issues
- Partial outage

**Response:**
1. Assess impact
2. Implement mitigation
3. Schedule fix

### SEV3 - Medium
- Minor feature issues
- Non-critical bugs

## Escalation

1. L1: On-call engineer
2. L2: Engineering lead
3. L3: CTO

## Communication

- Internal: #incidents Slack channel
- External: Status page updates
